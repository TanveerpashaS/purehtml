🌐 Lecture 1 — How Does The Internet Work?

         
            ▶️ Lecture 1: Core Concepts of the Internet            
             

The 5 Core Principles

             

The internet isn't magic. It's a series of logical solutions to fundamental problems.


           

The 5 Core Principles

The internet isn't magic. It's a series of logical solutions to fundamental problems.

  1. The Need to Share Information: At its core, we need to get data from Computer A to Computer B. The simplest solution is a physical cable, creating a direct link to send bits (1s and 0s).

  2. The Problem of Scale: Connecting every computer to every other is impossible. The solution is a central device (a Switch) that creates a Local Network.

  3. Connecting Networks: To connect different local networks, we use a special device called a Router. Its job is to pass traffic between networks.
    "The Internet is not a cloud; it is a 'network of networks' connected by routers."

  4. The Problem of Finding Anyone: To find a specific computer among millions, every device needs a unique address. This is the IP Address.

  5. The Problem of Reliability: Sending large files at once is risky. The solution is to break data into thousands of small, numbered Packets using a set of rules called TCP/IP.
             
             

IP Addresses, MAC Addresses, and Ports

             

A MAC Address is a permanent, physical serial number for your hardware. An IP Address is its address on the network. A Port Number is a door to a specific application on that device.

                                                                                           
Port TypeRangePurpose & Examples
Well-Known0 - 1023Standard services (Requires admin rights). Ex: 80 (HTTP), 443 (HTTPS), 22 (SSH).
Registered1024 - 49151Specific applications (No admin rights). Ex: 27017 (MongoDB), 3000 (React Dev).
Dynamic49152 - 65535Temporary ports assigned to your browser for outgoing connections. Not used for servers.
           
         
       
       
               
         

📝 Lecture 2 — HTML: HyperText Markup Language

         
            ▶️ Lecture 2: The Building Blocks of the Web            
             

What is HTML?

             

HTML is the standard language used to create and structure the content of a webpage. It uses "tags" to give meaning to the content.


                                                                                                                                                                   
HyperTextMarkup Language
Refers to links that connect web pages to one another, creating a "web" of information.Means you use "tags" to surround content, giving it meaning and structure (e.g., marking text as a heading).
             
             

Essential HTML Tags

             

The following are the foundational building blocks of any webpage.

             

Lists: <ul>, <ol>, <li>

             

Used to group related items. <ul> for bulleted lists, <ol> for numbered lists.

              

<ul>
  <li>Tea Bag</li>
</ul>
<ol>
  <li>First, boil the water.</li>
</ol>
             
             

Links (Anchor): <a> and Image: <img>

             

Creates a clickable hyperlink. Use target="_blank" to open in a new tab. The <img> tag embeds an image.

             
                             
           
         
       
       
               
         

📁 Lecture 3 — File Paths, Boilerplate, & Structure

         
            ▶️ Lecture 3: Organizing Your Project            
             

Part 1: The File Path System

             

A website is a collection of files in folders. A file path is the exact address the browser needs to find and link these files together.


             

Relative File Paths (For Your Own Project Files)

             

Directions to a file starting from your current location. Use these 99% of the time.

             

Scenario 1: Same Folder: Just use the filename: <a href="about.html">About</a>

             

Scenario 2: Into a Sub-Folder: Go down: <img src="images/logo.png">

             

Scenario 3: Out of a Folder: Go up, then down: <img src="../images/logo.png">

             
             

Part 2: The HTML Boilerplate

             

The boilerplate is a standard template that you must start every HTML file with. It gives the browser critical setup information before rendering your content.

              

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
</body>
</html>
             
             
The boilerplate is the non-negotiable foundation of any professional webpage.
             
             

Part 3: Page Structure & Semantics

             

Using tags with meaning (semantics) is better than using <div> for everything.

                                                                                           
Semantic TagPurpose
<header>Contains introductory content for a page or section, like a logo and navigation.
<main>Contains the main, unique content of the page. Should only be used once per page.
<footer>Contains closing content for a page or section, like copyright info and contact links.

🎬 Lecture 5 — HTML Media

         
            ▶️ Lecture 5: Native Video and Audio            
             

Beyond Text & Images

             
The web needed a standardized, native way to embed video and audio without insecure third-party plugins like Flash. The solution was the HTML5 <video> and <audio> tags.

             

Embedding Methods: Simple vs. Professional

             

The "Professional" method using <source> is recommended for public websites to ensure maximum browser compatibility.

                                                                                                                                                 
FeatureProfessional Method (<source>)Simple Method (src attribute)
Example

<video controls>
 <source src="vid.mp4">
 <source src="vid.webm">
</video>
                    

<video src="vid.mp4" controls></video>
                   
CompatibilityExcellent. Provides multiple formats as fallbacks.Good, but risky. Fails if the browser doesn't support the single format.
